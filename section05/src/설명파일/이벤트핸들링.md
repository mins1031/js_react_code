## 이벤트 핸들링

사용자의 모든 동작으로 인한 동작을 이벤트라고함.
더 넓게 보면 어플리케이션에 대한 모든 요청이 하나의 큰 이벤트라고 할수 있음

이벤트 핸들링은 이벤트를 처리하는것을 의미

- 이벤트 핸들링 코드

```
const Button = ({text, color = "black", children, url}) => {
    // 이벤트객체라는 것도 존재.
    const onClickButton = (e) => {
        console.log(e);
        console.log(text);
    }

    return (
        <button
            onClick={onClickButton}
            // onMouseEnter={onClickButton}
            // 위같은들을 이벤트 핸들러라고 함. 제공되는 여러 이벤트핸들러가 있나봄
        style={{color: color}} url={url}>
        {text} - {color.toUpperCase()}
        {children}
        </button>
    )
}
```

### 이벤트핸들링

버튼태그의 onClick, onMouseEnter 같은 리액트제공 함수들을 기반으로 이벤트 핸들링을 할수 있다고함.
그리고 이런 이벤트핸들링 함수들은 이벤트객체 라는것을 받을수 있는데
onClickButton 함수의 e 가 이벤트 객체로 콘솔에 출력된 내용을 보면
SyntheticBaseEvent라는 객체가 로깅됨 이 객체는 합성 이벤트라고 함

### 합성이벤트

합성이벤트란 모든 웹브라우저의 이번트 객체를 하나로 통일한 형태임.
프론트는 브라우저별로 스팩이달라서 발생하는 문제들이 많고 이걸 'Cross Browing Issue' 라고한다.
이 CBI를 해결하는 친구중하나가 '합성이벤트 객체'임. 일종의 통합규격이라고함.
그래서 이 합성이벤트객체를 잘 활용하면 CBI 를 크게 신경쓰지 않아도 된다고함.
