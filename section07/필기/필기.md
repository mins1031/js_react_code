## CSS 백그라운드 UI 를 적용하기 위한 태그

App.jsx에 Viewer,Controller 컴포넌트를 section 태그로 감싸줌.

이건 추후 CSS 적용시 컴포넌트별 백그라운드와 여백같은 내용을 적용하기 위함이라고함
일종의 구역을 나눠주는 역할이라고 이해하면 될듯?

- 관련 코드

```
<section>
    <Viewer />
</section>
<section>
    <Controller />
</section>
```

## CSS 적용을 위한 div 태그 처리

Css도 관리의 용이를 위해 역할별로 파일을 나눠놓음
위 예시 코드에서 아래 예시코드의 차이를 보면 컴포넌트들을 div 태그로 감싸고 div 태그엔 className이라는 스타일 값이 적용 되어있음

일단 설명은 import './App.css' 요 css 파일에서 className이 App인 요소내의 section 태그에만 일정한 스타일 처리를 할수 있음
즉 일종의 타겟팅을 하기 위한 작업이라고 이해하면될듯

```
import './App.css'
import Viewer from './components/Viewer'
import Controller from './components/Controller'

function App() {

  return (
    <div className="App">
    <h1>Simple Counter</h1>
    <section>
      <Viewer />
    </section>
    <section>
      <Controller />
    </section>
    </div>
  )
}

export default App
```

## 어떤 컴포넌트에 카운트 state를 적용해야할까?

카운트 값을 랜더링 하려면 state로 설정해야한다.
그리고 state는 컴포넌트에만 적용할 수 있다
그렇다면 어떤 컴포넌트에 적용해야 할까?

1. 부모인 App
2. 카운트를 노출하는 Viewer
3. 카운트 증감 버튼이 있는 Controller

정답은 APP 이라고한다.
2,3은 서로 부모 자식 관계의 컴포넌트가 아니기에 값을 공유할 방법 자체가 없다고함.

만약 Viewer 컴포넌트에 state 를 선언하게 되면 코드를 아래와 같다.
그리고 카운트를 증가시키기 위해선 stated의 setCount() 함수를 Controller 컴포넌트에 전달해야한다.
가능한가? -> x
리액트에선 컴포넌트간 데이터를 주고 받을때 Props를 사용하는데.. 이 Props는 부모-자식간의 관계에만 전달 가능함.-> 당연한게 선언하면서 전달해야하기 때문인데 Viewer에 state가 선언되어 있으면.. 어떻게 전달할건데?

만약 그 반대로 Controller에 생성하게되면 Viewer에 전달을 못하고 이렇게 되면 증감시켜놓고 UI에 노출하지를 못함.

## 카운트앱 state 적용하는 방법

조금 햇갈리긴함.

1. state 선언 처리.
2. Viewer에 props로 state 값을 단순히 넘겨주면됨. 이런 리랜더링 일어날때도 state값을 넘기기에 뷰어에 노출하는 기능은 이걸로 ok
3. 컨트롤러 컴포넌트에서 count 값을 변경하려면 APP에서 count, setCount()를 전달후 각 버튼에 맞게 setCount(count - 10) 이런 식으로 처리해야함
4. 3처럼 해도 되지만 이것보다 좋은 방식은 APP에서 이벤트 핸들러를 만들어 props로 전달하는 방법임.
5. 이벤트 핸들러인 onClickUpdate는 value라는 값을 전달받아 count의 값과 더해주는 역할임. 즉 해당 함수 호출시 해당 동작을 수행하는 파라미터 함수.
6. 이걸 Controller 컴포에 이벤트핸들러를 전달하면 이걸 받아서 버튼에 맞는 값(=value)을 핸들러의 동작으로 수행함.
7. 다만 Controller에서 왜 버튼 태그내에 onClick에 함수로 선언해서 전달할까?

- 그냥 <button onClick={onClickButton(-1)}> 이렇게 처리하면 onClick에 함수의 동작을 수행하게 하는게 아닌 함수의 결과값을 선언하게 되어 버림. 해당 함수는 리턴값이 따로 없기에 원하는 동작을 수행할 수 없음
  - 해봤는데.. 아예 화면 자체가 노출이 안됨..
- 또한 <button onClick={onClickButton}> 이렇게는 원하는 인수를 전달할수 없음
  - 얘도 화면 노출 x

- APP.jsx

```
import './App.css'
import Viewer from './components/Viewer'
import Controller from './components/Controller'

import { useState } from "react";

function App() {
  const [count, setCount] = useState(0);

  const onClickUpdate = (value) => {
    setCount(count + value);
  }

  return (
    <div className="App">
    <h1>Simple Counter</h1>
    <section>
      <Viewer count={count}/>
    </section>
    <section>
      <Controller onClickButton={onClickUpdate}/>
    </section>
    </div>
  )
}

export default App
```

- Controller.jsx

```
const Controller = ({onClickButton}) => {
    return (
        <div>
            <button onClick={() => {
                onClickButton(-1)
            }}>
                -1
            </button>
            <button>-10</button>
            <button>-100</button>
            <button>+100</button>
            <button>+10</button>
            <button>+1</button>
        </div>
    );
}

export default Controller;
```

## 이번 프로젝트의 중점사안

1. 여러 컴포넌트가 계층구조를 형성하면서 앱이 만들어짐
2. 특정 컴포가 다른 컴포에게 데이터를 전달하려면 반드시 두 컴포는 부모0-자식 관계여야함
3. 하나의 state를 여러 컴포에서 관리해야 할경우 이 state는 여러 컴포의 부모컴포에서 관리해야함
   이런 방식을 리액트에선 State Lifting(State 끌어올리기) 라고 표현한다
4. 결론적으로 리액트에선 Props 기능을 통해 부모에서 자식에게 전달할 수 있고 데이터들은 항상 위에서 아래로, 한 방향으로 흐르게 되고 이를 단방향 데이터 흐름이라고 부름.-> 파악하기 용이, 직관적임.
5. 결국 데이터의 원천인 state를 어떤 컴포에서 관리하고 정의할지 고민을 많이 해야한다.
