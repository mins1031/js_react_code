## 라이프 사이클

사람에게도 생애주기가 있듯이 리액트 컴포넌트에도 생애주기. 즉 라이프사이클이라는것이 있다고함

### 리액트 컴포넌트의 라이프 사이클

- 아래 3단계로 나뉜다고 한다.

1. Mount - 탄생
   - 컴포넌트가 탄생하는순간 = 화면에 처음 렌더링 되는 순간
   - "A 컴포넌틀가 Mount 되었다!" 라는 의미는 "A 컴포넌트가 화면에 처음으로 렌더링 되었다" 로 이해하면 된다.
2. Update - 변화
   - 컴포넌트가 다시 렌더링 되는 순간을 의미 = 즉 리렌더링 될 때를 의미한다
   - "A 컴포넌트가 업데이트 되었다" = "A 컴포넌트가 리렌더링 되었다."
3. unMount - 죽음
   - 컴포넌가 화면에서 사라지는 순간. = 렌더링에서 제외되는 순간을 의미
   - "A컴포넌트가 언마운트 되었다" = "A 컴포넌트가 화면에서 사라짐"

### 라이프 사이클 이해의 효용

- 라이프 사이클을 재대로 이해하고 있다면 특정행위들을 각 사이클에 맞게 적재적소에 배치할 수 있다.
- ex)
  - 서버에서 데이터를 불러오는 작업 -> 처음 화면이 호출될때 데이터도 함께 있어야함 -> 마운트 사이클에서 동작시켜야함
  - 화면에서 변경된 값을 콘솔에 출력하는 작업 -> 뭔가 변경되게 된다면 리렌더링이 일어나야함 -> 업데이트 사이클에서 동작
  - 컴포넌트가 종료될때 사용하던 메모리 정리 -> 당연히 더이상 해당 컴포넌트가 필요없는 상황 = 컴포넌트 렌더링이 종료되는 순간임으로 -> 언마운트 사이클에서 동작.
- 이렇게 특정행위들을 적절한 상황에 동작시키기 위한 개념이 라이프사이클이고 이것을 "라이프사이클 제어" 라고 한다
- 이 라이프사이클 제어 는 리엑트 훅중 하나인 "useEffect"를 사용하면 쉽게 할 수 있다고 함.
