## 검색기능 구현

- List 컴포넌트에 처리. 검색인풋에 글자를 입력하는 순간순간 마다 리랜더링이 되게 처리해야 익숙한 검색 처리가 발생
- 생각보다 리렌더링 비용이 비싸지 않은듯?
- 내가 처음에 생각한 동작과는 좀 다름. 약간 엔터를 쳐야 검색이 되는 기능을 생각했는데 그건 아니고 검색란에 글자를 입력하는 순간순간 해당값을 찾아냄 -> 이건 사용자가 모든 글자를 검색할 필요 없이 일부의 문자만 입력하고도 원하는 값을 찾아낼수 있는 사용자친화 UI 라고 할 수 있을듯.
- 그럼 결국 빠르게 결과를 보여줘야 하니 글자글자당 결과값을 가시해주는게 맞겠네. 음음.
- 참고로 원래 저런값은 값을 입력할때마다 리렌더링 되긴함..

1. 검색인풋에 값을 받아 따로 저장해서 아이템의 컨텐츠와 비교해야 하기 때문에 search state 저장.
2. 저장된 search와 todos를 비교해 content에 search가 포함된 것만 응답하는 함수 생성(자바 stream의 filter().toList() 생각하셈 ㅇㅇ)
3. 기존에 todos 기반으로 투두목록 뿌려주던 부분을 2 함수의 결과값으로 대체
4. 이제 검색창에 검색하면 검색대상과 부합하는 아이템만 남고 나머지는 invisible 처리됨.
5. 또한 영어 대소문자에 따라 검색결과가 바뀌는건 검색의 기능을 완전히 제공하지 못하기에 각 비교값에 .toLowerCase()함수 적용으로 대응

```
const List = ({todos}) => {
    const [search, setSearch] = useState(""); // 1

    const onChangeSearch = (e) => {
        setSearch(e.target.value);
    }

    // 2
    const getFilteredData = ()=>{
        if(search === "") {
            return todos;
        }

        return todos.filter((todo) =>
            // 5
             todo.content.toLowerCase().
                includes(search.toLowerCase())
        )// 문자열 include는 인자에 해당하는 문자가 있으면 true. 그래서 true 인것만 모아서 return.
    }

    // 2+
    const filteredDatas = getFilteredData();

    return (
    <div className="List">
        <h4>Todo List 🌱</h4>
        <input
            // onKeyDown={onKeydown}
            value={search}
            onChange={onChangeSearch}
            onplaceholder="검색어를 입력하세요"/>
        <div className="todos_wrapper">
            // 3
            {filteredDatas.map((todo)=> {
                // return <div>{todo.content}</div>;

                return <TodoItem key={todo.id} {...todo} />;

            })}
        </div>
    </div>
    );
}

export default List;
```
