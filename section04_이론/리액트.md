React.js란?
대규모 웹서비스 UI를 더 편하게 개발하기 위한 기술

리액트엔 기술적 특징들이 있고 해당 강의는 그 특징들을 자세히 설명할 예정.

1. 컴포넌트를 기반으로 UI를 표현
2. 화면 업데이트 구현이 쉽다
3. 화면 업데이트가 빠르게 처리된다.

위 내용이 어렵게 느껴질수 있음
편하게 훑어보고 완강하고 다시 들어보셈 ㅇㅇ.

- 특징1 : 컴포넌트를 기반으로 UI를 표현
  기본 UI가 header, main, footer 로 나눠져 있다고하면 각 js 파일내에 컴포넌트라는 개념으로 나눠서 사용할수 있다. = 모듈화
  그래서 비슷한 페이지 개발시 헤더, 푸터 등을 재사용 가능.

- 특징2 : 화면 업데이트 구현이 쉽다
  업데이트란 사용자의 행동(클릭, 드레그)에 다라 웹페이지가 스스로 모습을 바꿔 상호작용하는것.
  리액트는 이런 업데이트 구현이 쉬운데 그 이유는 선언형 프로그래밍으로 이뤄져 있기 때문
  선언형 프로그래밍이란. 과정을 생략하고 목적만 간결히 명시하는 방법 이라고 함.
  그러므로 목적만 명시되어 코드가 깔끔하고 간결해짐
  (반대의 개념은 명령형 프로그래밍이 있음)

그리고 이런 선언형 프로그래밍을 가능하게 하는 개념이 'State' 이다 state의 변수값에 따라 다른 UI를 노출해줄수도 있음.

- 특징3 : 화면 업데이트가 빠르게 처리된다.
  브라우저는 우리가 만든 HTML, CSS를 화면에 노출하기 위해 여러가지 과정을 처리하는데 이걸
  '브라우저의 랜더링 과정(Critical Rendering Path)' 라고 한다.
  (위 내용은 프론트 개발할때 계속 마주치는 개념이라고함.)

* DOM & CSSOM
  - HTML -> DOM : HTML파일 DOM 이라는 파일로 변환
  - CSS -> CSSOM : CSS파일 CSSOM 이라는 파일로 변환

여기서 DOM이라는건 HTML을 브라우저가 알기 쉽게 변환한 Document Object Model 이라고함. 즉 그냥 고수준언어(HTML)을 저수준언어(DOM)으로 변환했다고 보면 될듯(Java -> byte code)

- Render Tree 작업
  이후 Render Tree 라는것을 만듬. 이건 웹페이지의 청사진 같은 개념임.
  DOM 에는 요소들의 위치, 배치, 모양에 관한 정보가 존재할거고
  CSSOM 에는 요소들의 스타일과 관련된 정보가 존재할거임
  Render Tree 는 위 DOM, CSSOM을 합친 결과라고 보면될듯

- Layout 작업
  웹페이지 공간 안에 요소들의 위치를 배치를 정하는 작업.
  (내방에 가구들의 위치정하는걸 레이아웃 잡는다고 생각하면 이해가 편함.) + 다른 과정에 비해 오래걸림(=무거움)

- Painting
  실제로 웹페이지에 내용을 그려내는 과정.

* 다른 과정에 비해 오래걸림(=무거움)

그럼 업데이트는?
Js가 DOM 을 수정할때 일어남.
DOM 수정시 위의 랜더링 과정이 다시 일어남
DOM 수정 -> Render Tree -> Layout -> Painting

이때 업데이트로
Layout이 다시 실행되는 걸 Reflow
Painting 다시 실행되는 걸 Repaint
라고한다

그럼 js로 수정을 3000번 하게 되면? 위 과정이 3000번 수행.
특히 Reflow, repainting 이 무거운 작업이기에 부담스러움.
프론트도 성능이 중요하고 이 성능은 주로 업데이트와 관련이 있을듯?

그런데 위 최적화 과정을 React는 자동으로 해준다고함.
리액트는 내부적으로 Virtual DOM 이라는 기술을 사용한다.

- Virtual DOM : DOM 을 흉내낸 js 객체
- 리액트는 업데이트가 발생하면 실제 DOM 수정전 Virtual DOM에 먼저 적용시켜본다고 합니다. 그래서 모두 virtualDOM 에 적용후 적용 내용을 실제 DOM에 적용함으로 브라우저 랜더링 과정중 무거운 Reflow, Repainting 과정을 불필요하게 많이 하지 않아도되게 최적화 되어있는것. (일종의 버퍼역할)

## 첫 리액트앱 생성하기

리액트는 필요한 기본 라이브러리들이 너무 많다. 이걸 일일이 조사하고 의존성 넣어주고 하려면 굉장히 오랜 시간이 걸린다.
그래서 나온게 'vite' 리액트의 기본 라이브러리와 여러 설정들을 가지고 리액트 앱을 생성해주는 프레임워크? 라고 한다.
(약간 리액트 = 스프링, vite = 스프링부트 요 느낌이넹)
vite를 받으면 바로 리액트 디렉토리가 생성된다 하위의 디렉토리들은 어떤 의미이고 어떤 역할을 하는지 알아보자

- public
  해당 폴더 내에 vite.svg 파일이 있다. 해당 파일은 정적인 파일을 저장하는 파일이라고 볼수 있고, 사진, 영상 등의 내용을 추가할때 해당 파일에 추가한다고한다
- src
  실제 작성할 코드들을 보관하는 폴더.
  스프링이랑 똑같넹
  - assets
    해당 폴더 하위엔 react.svg가 존재한다. 위의 vite.svg와 역할은 같지만 용도가 살짝다르다는데 이건 뒤에서 배움. 일단 용도가 다른 정적파일 보관 폴더다 라는 것만 기억하자
- eslintrc.cjs
  eslintrc는 개발자들의 코드스타일 통일해주는 라이브러리라고함. 기본적으론 당장 몰라도된다고함.
- index.html
  간단한 html 코드가 존재. 리액트의 기본틀이라고함.
  이것도 그냥 그렇구나~ 하고 당장은 넘김
- vite.config.js
  이것도 설정파일. 뒤에서 배움
